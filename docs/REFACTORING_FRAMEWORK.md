## 全局设计决策（适用于所有页面）

### 布局方向：全宽优先

> 用户明确偏好：**整个网站不适合中间内容放太窄的风格，全宽会更好。**

这意味着：

| 原规范建议 | **新方向调整** |
|-----------|--------------|
| `.container` max-width: 1400px | **Hero 区域 → 100% 全宽（仅 padding 约束）** |
| `.container-narrow` max-width: 800px | **内容区 → 1400~1600px（充分利用屏幕）** |
| `.container-full` max-width: 100% | **表格/卡片网格 → 100% 全宽或接近全宽** |
| 表单区 800~1000px | **表单保持 900~1100px 居中（可读性底线）** |

#### 新容器体系

```css
/* ====== 全宽时代的容器策略 ====== */

/* 页面根容器 — 仅左右 padding，不限制宽度 */
.page-root {
  width: 100%;
  padding: 0 var(--space-10);    /* 左右各 40px 呼吸空间 */
  box-sizing: border-box;
}

/* Hero 区域 — 真正的全宽 */
.hero-full {
  width: 100%;
  padding: var(--space-16) var(--space-10);  /* 大量纵向留白 */
}

/* 内容区域 — 宽松但不无边 */
.content-wide {
  max-width: 1440px;             /* 比 1400px 稍宽 */
  margin: 0 auto;
  padding: 0 var(--space-6);     /* 内部再缩一点 */
}

/* 表格/数据网格 — 接近全宽 */
.data-full {
  max-width: 1600px;             /* 数据展示可以更宽 */
  margin: 0 auto;
}

/* 表单/阅读区 — 保持舒适行宽 */
.form-comfortable {
  max-width: 960px;              /* 表单不铺太宽，保证填写体验 */
  margin: 0 auto;
}
```

#### 断点策略（统一）

```css
/* 超大屏 ≥ 1600px — 充分展开 */
@media (min-width: 1600px) {
  .content-wide { max-width: 1560px; }
  .data-full   { max-width: 1760px; }
}

/* 标准大屏 1200~1599px — 主要区间 */
@media (min-width: 1200px) and (max-width: 1599px) {
  .content-wide { max-width: 1160px; }
  .data-full   { max-width: 1360px; }
}

/* 平板 768~1199px */
@media (min-width: 768px) and (max-width: 1199px) {
  .page-root   { padding: 0 var(--space-6); }
  .content-wide,
  .data-full   { max-width: 100%; }
}

/* 手机 < 768px */
@media (max-width: 767px) {
  .page-root   { padding: 0 var(--space-5); }
  .hero-full   { padding: var(--space-10) var(--space-5); }
}
```

  border: 1px solid var(--color-border);
  box-shadow: var(--shadow-sm);
  background: var(--color-surface);

:deep(.el-card__header) {
  border-bottom-color: var(--color-border-light);
  padding: var(--space-5) var(--space-6);

/* 2. 输入框 */
:deep(.el-input__wrapper) {
  border-radius: var(--radius-md);
  box-shadow: 0 0 0 1px var(--color-border) inset;
}
:deep(.el-input__wrapper:hover) {
  box-shadow: 0 0 0 1px var(--color-solid) inset;
}
:deep(.el-input__wrapper.is-focus) {
  box-shadow: 0 0 0 1px var(--color-solid) inset,
              0 0 0 3px rgba(15, 23, 42, 0.08);

/* 3. 按钮 */
:deep(.el-button--primary) {
  background: var(--color-solid);
  border-color: var(--color-solid);
  border-radius: var(--radius-lg);
}
:deep(.el-button--default) {
  border-color: var(--color-border);
  color: var(--color-text-primary);
  border-radius: var(--radius-lg);
}

## 通用重构 Checklist（每页必做）

### 第一遍：结构性改动
- [ ] 页面根容器宽度更新为推荐值
- [ ] Hero 区域改为 `hero-full`（全宽 + padding）
- [ ] 内容区域使用对应的容器类（content-wide / data-full / form-comfortable）
- [ ] 响应式断点统一为 1600/1200/768 三档
- [ ] 表单区域独立收窄居中（如适用）

### 第二遍：CSS 变量化
- [ ] 搜索 `#[0-9a-fA-F]{3,6}` 和 `rgba(` 硬编码颜色 → 替换为 CSS 变量
- [ ] 搜索 `box-shadow` 硬编码 → 替换为 `--shadow-*` 变量
- [ ] 搜索 `border-radius: \d+px` → 替换为 `--radius-*` 变量
- [ ] 搜索 `font-size: \d+px` → 替换为 `--text-*` 变量
- [ ] 搜索 `padding`/`margin`/`gap` 硬编码 → 替换为 `--space-*` 变量

### 第三遍：Element Plus 样式统一
- [ ] `:deep(.el-card)` → 统一圆角/边框/阴影
- [ ] `:deep(.el-input)` → 统一边框/聚焦环
- [ ] `:deep(.el-button)` → 统一 primary/default 颜色
- [ ] `:deep(.el-tabs)` → 统一 active-bar 颜色
- [ ] `:deep(.el-tag)` → 统一语义色映射
- [ ] `:deep(.el-radio-button)` / `:deep(.el-checkbox-button)` → 统一选中态

### 第四遍：Content Zone 动效审查
- [ ] 检查是否有 shimmer/pulse/flow 动画 → 移除
- [ ] 检查 `transition` 是否过度花哨 → 简化为 `var(--transition-fast)`
- [ ] 检查 hover `transform` 幅度 → Content Zone 不超过 `translateY(-4px)`
- [ ] 检查 `box-shadow` hover 是否多层发光 → 改为单层 `--shadow-lg`

### 第五遍：视觉走查
- [ ] L1 焦点是否唯一且明显？（标题/关键数据）
- [ ] L2 支持信息是否 2~4 个成组？
- [ ] L3 辅助细节是否不抢注意力？
- [ ] 密度变化是否有节奏？（不是全程均匀）
- [ ] 留白是否起到分组/强调作用？
