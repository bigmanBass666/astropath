## 一、站点概述

**AstroPath（智途）** 是一个面向中国学生的智能留学规划平台。它通过系统化的个人背景评估、大数据驱动的院校匹配算法，以及 AI 智能对话，为留学申请者提供从"认识自己"到"做出选择"的一站式服务。

当前版本包含以下核心功能模块：

| 模块 | 功能 |
|------|------|
| 首页 | 品牌展示与功能导航，带动画粒子背景 |
| 背景评估 | 多维度个人信息采集与可视化竞争力分析 |
| 选校推荐 | AI 智能选校建议 + 本地算法双引擎推荐 |
| 时间规划 | 分阶段留学时间轴与任务追踪 |
| 材料中心 | 文书模板、清单管理与 AI 写作助手 |
| 院校数据库 | 55+ 所世界大学详细档案检索 |
| AI 对话 | 4 个专业智能体在线咨询服务 |
| 院校/专业详情 | 单校单专业深度介绍页 |

## 二、设计理念

### 2.1 为什么做这个项目

留学规划是一个信息密集且决策复杂的过程。学生需要在海量院校信息中找到适合自己的目标，却往往缺乏系统性的工具来整合信息、评估自身条件和获取即时建议。

AstroPath 的核心理念是：**让每个留学申请者都能拥有一个"数字化顾问"**——它了解你的背景，能给出个性化的建议，并且随时待命。

### 2.2 设计美学

平台采用 **Slate + Amber 极简学术风**设计体系——以 Slate（岩灰）单色系构建视觉层次，仅用 Amber（琥珀色）作为关键交互点缀。这种设计参考了 Apple、Stripe、Linear 等现代产品的极简哲学，克制而不失温度，既体现学术严肃性又不显冰冷。

页面还设计了**两级氛围系统**：表单类页面使用轻量的网格背景保持干练，数据展示类页面叠加径向渐变营造纵深感，让整个平台在统一中富有变化。

## 三、功能详解

### 3.1 背景评估系统

这是用户进入平台后的第一步——通过一份全面的多步骤表单，系统从五个维度收集用户信息：

- **基础信息**：姓名、年龄、院校层次（985/211/海外/普通）、GPA、语言成绩
- **学术背景**：目标学位、意向专业、均分、科研经历
- **实践背景**：实习经历、竞赛获奖、志愿服务

评估完成后，系统会生成一张 **ECharts 五维雷达图**（学术能力、语言成绩、科研经历、实习实践、综合素质），让用户一目了然地看到自己在竞争力坐标系中的位置。

### 3.2 AI 智能选校推荐

这是整个平台最核心的功能模块。其工作流程如下：

**第一步：偏好采集**
用户通过交互式表单明确自己的选校偏好，包括优先级排序、排除国家、特殊需求等。

**第二步：AI 深度分析**
系统会将用户画像数据与内置的 55 所大学数据库组合，生成结构化的 AI Prompt 提交给大语言模型，得到个性化的选校分析报告。

**第三步：本地算法兜底**
如果 AI 服务暂时不可用，系统自动切换到本地匹配引擎。该引擎内置了一套评分算法，综合考虑用户 GPA 层次、语言成绩、科研权重、实践背景等因素，为每所学校计算匹配度分数，并划分为三个梯队：

- **Reach（冲刺）**：有挑战但有希望
- **Match（匹配）**：条件吻合，录取概率较高
- **Safe（保底）**：条件优于录取标准

**第四步：可视化展示**
每所推荐学校都有专属卡片，展示排名、匹配度、截止时间、学费等关键信息。用户可以通过 ConstellationCanvas（星座雷达图）直观对比多个维度的得分，CountUp 组件则以动画数字增强数据的易读性。

### 3.3 时间规划

以时间轴的形式呈现留学申请的完整流程，分阶段展示每个阶段需要完成的任务：

- 准备阶段：语言考试、科研积累
- 申请阶段：文书撰写、材料准备、网申提交
- 录取阶段：Offer 比较、签证申请

滚动页面时，关键数据会通过动画计数器动态展示进度。每个阶段可展开查看详情，完成状态实时追踪。

### 3.4 材料中心

为申请者打造的文书管理中心，包含：

- **文书模板库**：覆盖个人陈述、推荐信、简历等常见文书类型
- **清单管理**：申请所需材料的 Check-list 系统，完成状态追踪
- **AI 写作助手**：用户可以直接向 AI 咨询文书写作问题，获得实时流式回复
- **PDF 导出**：一键将分析报告导出为 PDF 文件，方便保存和打印

### 3.5 AI 对话系统

平台内置了 **4 个专业智能体**，各司其职：

| 智能体 | 角色 | 职责 |
|--------|------|------|
| 规划顾问 | Consultant | 整体留学规划、国家选择、策略制定 |
| 文书专家 | Essay | 文书写作指导、润色建议、范例参考 |
| 选校顾问 | Selection | 选校分析、Reach/Match/Safe 建议 |
| 签证专家 | Visa | 各国签证流程、材料准备、注意事项 |

AI 对话支持 Markdown 格式渲染和**思维链展示**——用户可以看到 AI 的推理过程，也可以隐藏以保持界面简洁。所有对话历史保存在本地，随时可回溯查阅。

### 3.6 院校数据库

内置 55 所世界大学详实档案，覆盖美国、英国、香港、澳大利亚、新加坡、加拿大、日本、欧洲等主流留学目的地。每所学校包含：

- QS 排名、学费、录取率、申请截止时间
- 录取要求（GPA、语言、推荐信、科研要求等）
- 项目信息、学校简介
- 参考来源链接（官方数据，有据可查）

支持按国家、地区、排名区间等多条件筛选检索。

## 四、技术架构

### 4.1 整体架构

AstroPath 基于 **Vue 3 框架构建**，采用现代前端工程的组件化开发模式，具有以下特点：

- **纯前端应用**：无需后端服务器，所有数据通过浏览器本地存储（localStorage）持久化
- **响应式设计**：页面自适应桌面端和移动端
- **路由懒加载**：每个页面按需加载，减少首屏等待时间
- **构建工具**：使用 Vite 进行开发构建，支持热更新，开发体验流畅

### 4.2 技术选型

| 技术类别 | 选用方案 | 说明 |
|---------|---------|------|
| 前端框架 | Vue 3 | 当前主流的前端框架之一，响应式数据绑定 |
| 组件库 | Element Plus | 成熟的桌面端 UI 组件库 |
| 图表可视化 | ECharts | 百度开源的数据可视化库 |
| 路由管理 | Vue Router | Vue 官方路由 |
| Markdown 渲染 | Marked + DOMPurify | 渲染 Markdown 文本并过滤不安全内容 |
| PDF 导出 | html2canvas + jsPDF | 将网页内容生成 PDF 文件 |
| 代码质量 | ESLint + TypeScript 类型检查 | 提交前自动检查 |

### 4.3 核心技术：AI 流式通信系统

平台最核心的技术创新在于其自研的 **AI 流式响应处理系统**。这是整个平台"智能"能力的底层引擎。

**多供应商适配层**

不同 AI 供应商（OpenAI、Anthropic Claude、DeepSeek 等）的 API 格式各不相同。系统构建了一个统一的抽象层——无论用户配置的是哪种 AI 供应商，上层调用方式完全一致。

**思维链分离渲染**

新一代 AI 模型（如 Claude）在给出正式答案前会先有一段"思考过程"。系统能够自动识别并分离这两类内容：思考过程默认折叠存放，正式正文正常展示，用户可以随时切换查看。

**智能重试与降级**

当 AI 服务连接不稳定时，系统会自动尝试重连，每次等待时间逐渐加长（指数退避策略）。如果 AI 推荐服务完全失败，系统无缝切换到内置的本地匹配算法，确保用户永远能看到结果。

**任务队列管理**

系统维护一个任务调度器：当多个 AI 请求同时发出时，系统按优先级排队处理，最多同时执行 3 个任务，其余进入队列等待。

**滚动智能检测**

在流式输出过程中，系统会检测用户的滚动行为——如果用户向上滚动查看历史内容，自动滚动会暂停以免打断用户阅读；当用户滚回底部时，自动恢复。

---

### 4.4 数据安全与隐私

- 所有用户数据（个人信息、对话记录、配置信息）均保存在用户自己的浏览器本地，不上传任何服务器
- AI 生成的 HTML 内容在显示前通过 DOMPurify 库进行安全过滤，防止恶意代码注入

## 五、数据库与内容

### 5.1 院校数据

平台内置了 **55 所世界大学**的详尽档案，每所学校包含 20 余个数据字段：

```
学校基本信息 ── 名称、国家、类型、学生人数、简介
录取数据 ── 排名、录取率、学费、截止时间
录取要求 ── GPA要求、语言要求、推荐信数量、科研要求
匹配算法参数 ── 最低GPA、偏好GPA、语言分数要求、院校层次认可、科研权重
参考来源 ── 排名来源、学费来源、录取率来源、要求来源（均附原始链接）
```

所有参考数据均来自官方公开渠道，可在院校详情页面查看原始链接。

### 5.2 专业数据

平台收录了 **12 个热门留学专业**的详细信息：

计算机科学、人工智能、数据科学、金融工程、工商管理(MBA)、公共政策、生物医学工程、新闻学、建筑学、统计学、心理学、英国文学

每个专业包括课程安排、职业发展方向、国内外薪资范围及参考来源。

## 六、创新点总结

| 创新维度 | 具体体现 |
|---------|---------|
| **AI 深度融合** | AI 不是装饰，而是渗透到每个核心功能——选校推荐、背景评估分析、文书指导、实时咨询 |
| **双引擎推荐** | AI 智能推荐与本地匹配算法互为备份，确保服务的鲁棒性 |
| **数据可溯源** | 每所学校的排名、学费、录取率等数据均标注原始来源链接，不凭空捏造 |
| **隐私优先** | 零后端、纯客户端，用户数据不离浏览器 |
| **沉浸式体验** | Canvas 动画、滚动触发、流式打字机效果，打造有温度的交互 |
| **多供应商支持** | 不绑定任何单一 AI 供应商，用户自由选择配置 |

## 八、项目运行方式

### 8.1 环境要求

- **操作系统**：Windows 7 / 10 / 11、macOS、Linux 均可
- **Node.js**：版本 18 及以上（请从 <https://nodejs.org/> 下载并安装）
- **浏览器**：Chrome / Edge / Firefox 等现代浏览器

### 8.2 Node.js 安装

1. 访问 <https://nodejs.org/> 下载 LTS（长期支持）版本
2. 运行安装程序，按提示完成安装
3. 打开命令提示符（cmd）或终端，输入 `node -v` 验证安装
4. 如显示版本号（如 v20.x.x），则表示安装成功

### 8.3 项目安装与运行

```bash
# 1. 打开命令提示符或终端，进入项目目录
cd 网页大赛+刘家暄+吴雨琳＋15627172859

# 2. 安装项目依赖（首次运行只需执行一次）
npm install

# 3. 启动开发服务器
npm run dev

# 4. 启动成功后，在浏览器地址栏输入以下地址访问：
#    http://localhost:3000
```

### 8.4 AI 功能使用

进入平台后，需先在「AI配置」页面添加一个 AI 模型（支持 OpenAI、Anthropic Claude、DeepSeek 等），配置成功后即可使用全部 AI 功能。
