## ⚠️ 踩坑经验库

### 1. Hero 全屏高度计算陷阱（重要！）

**问题**: 在 Vue 页面中设置 Hero 区域为全屏时，直接用 `100vh`/`100dvh` 会导致：
- 内容溢出一屏（底部露出下方内容区）
- 或者不够填满一屏（留白）

**根因**: 本项目的页面被包裹在 App.vue 的 `el-container` 布局中：

```html
<!-- App.vue 结构 -->
<el-container class="app-container">
  <el-header>           <!-- ~64px 固定导航栏 -->
  <el-main class="app-main">   <!-- padding: 20px -->
    <router-view />      <!-- 你的页面在这里 -->
  </el-main>
</el-container>
```

**正确做法**:

```css
/* ✅ 正确：减去导航栏高度 + 抵消父级 padding */
.ud-hero {
  height: calc(100dvh - 64px);  /* 减去 el-header 高度 */
  margin-top: -20px;            /* 抵消 app-main 的 padding: 20px */
}

/* 响应式同步调整 */
@media (max-width: 1199px) {
  .ud-hero { height: calc(100dvh - 56px); margin-top: -14px; }
}
@media (max-width: 767px) {
  .ud-hero { height: calc(100dvh - 52px); margin-top: -10px; }
}
```

**通用规则**: 当页面需要全屏 Hero 时，必须同时考虑：
1. `el-header` 的实际高度（约 60-64px）
2. `app-main` 的 `padding`（默认 20px）
3. 使用 `100dvh` 而非 `100vh`（动态视口高度，适配移动端地址栏）

### 2. Awwwards 设计风格规范

本项目的院校库页面采用 Awwwards 级别的设计语言，后续页面重构需遵循：

- **BEM 命名**: 统一 `ud-` 前缀（如 `.ud-card__accent--danger`）
- **等宽数据字体**: 数字、标签用 `var(--font-family-mono)` (JetBrains Mono)
- **噪点纹理**: 全页 SVG fractalNoise 叠加（opacity 0.03）
- **动画曲线**: 
  - 入场: `cubic-bezier(0.16, 1, 0.3, 1)` (Expo Out)
  - 悬停微交互: `cubic-bezier(0.34, 1.56, 0.64, 1)` (Back Out)
- **筛选芯片**: 激活态用边框强调而非实色填充（`rgba(15,23,42,0.06)` 底色）
- **卡片悬停**: 3D perspective 倾斜效果（CSS `--mx/--my` 变量驱动）
- **禁止**: Inter/Roboto 字体、蓝白配色、三栏 features section、渐变 hero banner

### 3. 减少动画偏好支持

所有动画页面必须包含：

```css
@media (prefers-reduced-motion: reduce) {
  /* 禁用所有 transform 动画 */
  /* 禁用所有 animation */
  /* transition 缩短到 0.01ms */
}
```
