# 设计原则交接文档

> 本文档提炼 Assessment 页面重构的两个核心成功实践，供其他 AI 在其他页面重构时遵循。

---

## 核心实践一：全屏沉浸式 (Full-Screen Immersive)

### 原则
**打破版心限制，让内容占据整个视口**

### 具体要求
1. **移除版心卡片**
   - 删除 `max-width: 1200px`、`margin: 0 auto` 等限制
   - 删除"大屏中间小屏"的容器思维

2. **全屏场景 (Scene-Based)**
   ```
   每个功能模块 = 一个全屏场景
   
   Scene 0: Hero/入口 - 100vw × 100vh
   Scene 1: 功能A    - 100vw × 100vh
   Scene 2: 功能B    - 100vw × 100vh
   ...
   ```

3. **场景切换**
   - 通过导航点、按钮或滚动切换场景
   - 场景间使用平滑过渡动画

### 参考实现
- 文件：`src/views/Assessment.vue`
- 关键类：`.scene`、`.scene.is-active`

## 核心实践二：设计与功能有机融合

### 原则
**不要为了装饰而装饰，每个视觉元素都应该服务于功能**

### 具体要求

#### 1. 功能即视觉
- ❌ 错误：先做完功能，再套个好看的壳
- ✅ 正确：功能本身就应该以视觉化的方式呈现

**例子**：
```
GPA 输入 → 不应该只是输入框
        → 应该是可拖拽的环形进度条
        → 用户拖拽时实时看到数值和评级变化
```

#### 2. 数据可视化
- 数字 → 用大字体、醒目的方式展示
- 进度 → 用进度条、环形图展示
- 对比 → 用并排布局、雷达图展示

#### 3. 交互即反馈
- 每个操作都要有视觉反馈
- 悬停、点击、加载状态都要有对应的视觉变化

#### 4. 信息层次
- 重要的信息：大、醒目、居中或黄金分割位置
- 次要信息：小、淡化、辅助位置
- 引导用户视线流动

### 参考实现
- 文件：`src/views/Assessment.vue`
- 关键组件：
  - Hero 场景的标题层次（badge → title → subtitle → CTA）
  - GPA 环形可视化
  - Score Hero 卡片的数据展示
  - Dashboard 网格的信息组织

## 自由发挥空间

以下元素**不强制规定**，由各 AI 根据页面特性自由发挥：

| 元素 | 自由度 |
|------|--------|
| 背景效果 | 粒子/渐变/图片/视频/纯色均可 |
| 粒子形状 | 圆形/方形/自定义图形均可 |
| 动画细节 | 入场方式、时长、缓动曲线均可 |
| 布局细节 | 对称/非对称、网格/Flex 均可 |
| 微交互形式 | 磁吸/光晕/缩放/位移均可 |
| 字体选择 | 遵循项目字体规范即可 |

**唯一限制**：
- 配色必须遵循 `Slate-900 + Amber-600`
- 不能引入新的主色调

## 与 AI 沟通模板

```
请将 [页面名称] 重构，遵循以下两个核心原则：

【原则一：全屏沉浸式】
- 移除版心卡片限制
- 将页面按功能分割为多个全屏场景(Scene)
- 每个场景占据 100vw × 100vh
- 场景间可切换

【原则二：设计与功能有机融合】
- 功能本身要视觉化呈现，不要"功能+装饰壳"
- 数据要可视化展示
- 每个操作都要有视觉反馈
- 信息要有清晰的层次

参考实现：src/views/Assessment.vue

配色限制：Slate-900 (#0F172A) + Amber-600 (#D97706)
其他设计细节（背景效果、动画形式、布局方式等）请自由发挥。

页面功能场景：
- Scene 0: [功能描述]
- Scene 1: [功能描述]
- ...

请确保原有功能完整保留。
```

## 关键认知

**好的设计不是"好看的皮肤"，而是"功能的自然延伸"。**

当用户看到一个元素时：
1. 他应该立刻明白这是什么功能
2. 他应该感受到操作的反馈
3. 他应该被引导到下一步

视觉不是装饰，是功能的表达。
