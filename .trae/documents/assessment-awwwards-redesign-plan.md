# Assessment 页面 Awwwards 级重构计划

## 当前问题诊断

### 1. 版心布局问题（用户核心痛点）
- **问题**：当前卡片采用"大屏中间强行放置小屏"的版心设计
- **症状**：周围全屏沉浸式，但核心内容被圈住，丢失美学和设计感
- **根本原因**：`deck-card` 容器限制了内容宽度，形成尴尬的"画框效应"

### 2. 报告页面堆砌问题
- **问题**：AI结果页功能堆砌，缺乏设计美学
- **症状**：Score → Radar → Dimensions → AI Panel 线性堆叠，像功能清单
- **根本原因**：缺乏叙事节奏和视觉层次，没有"Wow Moment"

### 3. 表单卡片缺乏沉浸感
- **问题**：身份信息、学术背景、实践经历三张表单卡片视觉单调
- **症状**：白色背景 + 简单表单元素，没有Awwwards级别的视觉冲击力

---

## 重构策略：从"卡片堆叠"到"沉浸式叙事旅程"

### 核心理念
将 Assessment 从"填写表单"转变为"探索自我留学潜力"的沉浸式体验。每一屏都是一个**场景（Scene）**，而非**卡片（Card）**。

### 视觉语言定位
- **风格**：新极简主义 + 档案式排版 + 微妙的数据可视化美学
- **色彩**：严格遵循项目配色 `#0F172A` (Slate-900) + `#D97706` (Amber-600) + 纯白
- **字体**：JetBrains Mono (数据/标签) + 系统无衬线 (正文)
- **动效哲学**：每一次滚动/切换都是"揭开故事的下一页"

## 技术实现要点

### 1. 布局系统重构
```css
/* 放弃版心，使用全屏场景 */
.scene {
  min-height: 100dvh;

  flex-direction: column;
  justify-content: center;

/* 内容最大宽度限制，但视觉上无边框感 */
.scene-content {
  max-width: 1200px;
  margin: 0 auto;
  width: 100%;
}
```

### 2. 动画系统
```css
/* 场景切换动画 */
.scene-enter {
  animation: sceneEnter 0.8s cubic-bezier(0.16, 1, 0.3, 1) forwards;
}

@keyframes sceneEnter {
  from {
    opacity: 0;
    transform: translateY(40px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

/* 元素依次淡入 */
.stagger-item {
  opacity: 0;
  animation: fadeUp 0.6s cubic-bezier(0.16, 1, 0.3, 1) forwards;
}

.stagger-item:nth-child(1) { animation-delay: 0.1s; }
.stagger-item:nth-child(2) { animation-delay: 0.2s; }
```
