# Assessment 页面架构重构 v2 — 滚动沉浸式 + AI报告Reveal Moment

> **核心决策**: 确立「滚动沉浸式」为唯一导航模式，AI报告区重设计为「渐进式揭示」体验
> **方案**: C — Scroll + Contextual CTA + Report Reveal Moment

---

## 一、问题诊断

### 1.1 导航模式自相矛盾（截图确认）

```
当前状态:
┌─────────────────────────────────┐
│ Chapter 1: 身份信息              │ ← 滚动可达
│   [姓名输入] [数据芯片] [GPA环] │
│   [ 继续学术背景 → ]  ❌ 冗余    │ ← 滚一下就到了，按钮无意义
├─────────────────────────────────┤
│ Chapter 2: 学术背景              │ ← 滚动可达
│   [学历Pill] [标签云] [科研]     │
│   [ 下一步:实践经历 → ]  ❌      │
│   [ ← 返回 ]                ❌ │
├─────────────────────────────────┤
│ Chapter 3: 实践经历              │ ← 滚动可达
│   [三轨时间线]                  │
│   [ ← 返回 ] [生成报告 →] ⚠️   │ ← 唯一有意义的CTA
├─────────────────────────────────┤
│ Chapter 4: 报告 (混乱)           │
│   [← 返回] [模型选择] [生成报告]  │ ← 向导残留，布局崩坏
│   ┌──────┬──────────┐          │
│   │分数卡  │ AI分析面板│          │ ← 过度拥挤
│   │(大黑块)│(巨长文本) │          │
│   └──────┴──────────┘          │
│   雷达图 / 维度条 / 按钮        │ ← 被推到下面
└─────────────────────────────────┘
         ↑ Progress Orbit (多余?)
```

**矛盾**: 用户可以自由滚动浏览所有章节，但"下一步"按钮暗示必须点击才能前进。两种导航语义冲突。

### 1.2 AI报告区的具体设计缺陷

| # | 缺陷 | 截图证据 |
|---|------|---------|
| D1 | **向导残留** | "返回"+模型选择+生成报告 横排顶部 |
| D2 | **信息密度爆炸** | Bento+AI面板+雷达+维度条 同屏堆砌 |
| D3 | **AI文本撑爆** | 思考过程/分析报告 各数百字，纵向溢出 |
| D4 | **视觉层次丢失** | 前3章有呼吸感，第4章像Excel报表 |
| D5 | **操作按钮错位** | "保存/重新填写" 大按钮在底部，与滚动模式不协调 |

## 二、架构决策：方案C — 滚动沉浸式 + Reveal Moment

### 2.1 导航模式定义

```
新架构:

┌──────────────────────────────────────────┐
│  HERO (全屏入口)                          │
│  ↓ 渐变融合                               │
├──────────────────────────────────────────┤
│  ═══ CHAPTER 1: 身份 ═══                 │  ← 自然滚动到达
│  · 姓名 / 数据芯片 / GPA环 / 语言         │
│  · 无"下一步"按钮                        │
│  · 填写后自然向下滚                      │
├──────────────────────────────────────────┤
│  ═══ CHAPTER 2: 学术 ═══                 │  ← 自然滚动到达
│  · 学历Pill / 标签云 / 均分 / 科研时间线  │
│  · 无导航按钮                            │
├──────────────────────────────────────────┤
│  ═══ CHAPTER 3: 经历 ═══                 │  ← 自然滚动到达
│  · 三轨时间线 (实习/竞赛/志愿)            │
│  · 无导航按钮                            │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │  🤖 生成 AI 分析报告             │   │  ← 全页唯一的CTA
│  │     基于你的背景智能生成留学路线图   │   │
│  └─────────────────────────────────┘   │
├──────────────────────────────────────────┤
│  ═══ CHAPTER 4: 报告 (Reveal Moment) ══ │  ← 点击CTA后触发
│                                         │
│  阶段① 分数揭晓 (戏剧性数字)            │
│       ↓ 自动推进                        │
│  阶段② 雷达图辐射展开                    │
│       ↓ 自动推进                        │
│  阶段③ 维度条交错入场                    │
│       ↓ 完成                           │
│  阶段④ AI分析 (默认折叠)               │
│                                         │
│                              [💾 保存]  │  ← 浮动操作
├──────────────────────────────────────────┤
│  🔄 重新填写                             │  ← 底部微妙链接
└──────────────────────────────────────────┘

  ○ 身份  ← Progress Orbit(纯装饰+跳转)
  │
  ○ 学术
  │
  ○ 经历
  │
  ● 报告  ← 高亮当激活
```

### 2.2 关键设计原则

| 原则 | 说明 |
|------|------|
| **单一CTA** | 整个页面只有"生成AI报告"一个向导操作，位于Chapter 3末尾 |
| **零冗余导航** | 移除所有"下一步/返回"按钮，滚动是唯一的章节切换方式 |
| **Orbit降级** | Progress Orbit从"导航控制器"降级为"位置指示器+快速跳转" |
| **Reveal Moment** | AI报告不是静态表单，而是一个分阶段揭示的"仪式感"体验 |
| **渐进式披露** | 信息按重要性分层展示：分数 > 雷达 > 维度条 > AI详情 |

## 三、AI报告区 — Reveal Moment 详细设计

### 3.1 四阶段揭示流程

#### 阶段①: 分数揭晓 (Score Reveal)
**触发条件**: 用户点击"生成AI分析报告"后立即

**视觉效果**:
- 页面自动滚动到Chapter 4区域
- 显示一个居中的大型分数卡片（类似Hero标题的排版级别）
- 分数数字从0滚动到目标值（已有`animateScore`逻辑）
- 背景显示微妙的脉冲动画表示"AI思考中"
- 此时**只显示分数**，其他内容隐藏

```
┌─────────────────────────────────────┐
│                                     │
│            3.8                       │  ← 大号数字
│           / 5.0                      │
│                                     │
│     竞争力总分                         │  ← 小字标签
│                                     │
│   ▮▮▮  AI 正在深度分析...             │  ← 脉冲动画
│                                     │
└─────────────────────────────────────┘
```

#### 阶段②: 雷达展开 (Radar Unfolds)
**触发条件**: 分数稳定后（或AI stream开始输出content时）

**视觉效果**:
- 分数卡片缩小并左移到最终位置
- 右侧/下方出现雷达图容器
- 雷达图从中心向外辐射绘制（ECharts自带动画）
- 5个数据点依次亮起

```
┌──────────────────┬──────────────────┐
│                  │                  │
│    3.8 / 5.0    │    ○ 雷达图      │
│    竞争力总分     │   (辐射动画中)     │
│                  │                  │
└──────────────────┴──────────────────┘
```

#### 阶段③: 维度入场 (Detail Bars Stagger)
**触发条件**: 雷达图绘制完成后

**视觉效果**:
- 雷达图下方出现4个维度条
- 从上到下依次滑入（stagger 100ms间隔）
- 每条显示：名称 + 数值 + 进度条 + 一行评语

```
┌─────────────────────────────────────┐
│  学术能力  ████████░░  4.2 / 5      │  ← 第1个滑入
│  学术背景非常优秀，申请优势明显        │
│                                     │
│  语言能力  ██████░░░░  3.5 / 5      │  ← 第2个滑入
│  基本达标，争取更高分数              │
│                                     │
│  科研经历  ███░░░░░░░  1.5 / 5      │  ← 第3个滑入
│  建议增加科研经历以提升竞争力          │
│                                     │
│  实践背景  ██░░░░░░░░  1.2 / 5      │  ← 第4个滑入
│  空白，强烈建议补充                   │
└─────────────────────────────────────┘
```

#### 阶段④: AI分析面板 (AI Panel - Collapsible)
**触发条件**: 维度条全部入场完成后，或AI内容开始流式输出时

**视觉效果**:
- 出现在维度条下方的折叠面板
- **默认状态: 收起**，显示为一行:
  ```
  🤖 AI 分析报告已生成  ▼ 展开`
  ```
- **展开后**: 两个子标签页
  - 「思考过程」— 显示 reasoning 内容
  - 「分析报告」— 显示 AI content（Markdown渲染）
- 流式输出时显示打字光标和暂停/停止控制
- 面板使用与Identity Card类似的卡片风格（圆角、边框、微妙阴影）

### 3.2 操作按钮重新定位

| 按钮 | 当前位置 | 新位置 | 样式 |
|------|---------|--------|------|
| **保存结果** | 底部大按钮 | 右下角浮动按钮(FAB) | 圆形，图标为主，`position:fixed; bottom:32px; right:32px` |
| **重新填写** | 底部大按钮 | 页面最底部小字链接 | `color:#94A3B8; font-size:13px;` 居中 |
| **生成报告** | Chapter 3 顶部 | Chapter 3 末尾全宽CTA | 保持现有大按钮样式，但文案改为更有仪式感的表述 |

### 3.3 移除的元素

| 元素 | 原因 |
|------|------|
| Chapter 1-3 的 `chapter-action` 区块（含下一步/返回按钮） | 与滚动模式冲突，造成困惑 |
| Chapter 4 顶部的 `chapter-action--split`（含返回+模型选择+生成报告） | 向导残留；模型选择移入CTA内或移除 |
| `prevStep()` 函数调用 | 不再需要返回导航 |

## 四、Progress Orbit 重新定位

### 4.1 新职责

| 旧职责 | 新职责 |
|--------|--------|
| 控制章节显示/隐藏 | **纯装饰性位置指示器** |
| 必须点击才能前进 | **可选的快速跳转辅助** |
| active/done 状态驱动UI | **仅反映当前位置**(activeChapter) |
| 未到章节 = 不可见 | **所有章节始终可见** |

### 4.2 视觉调整

- 节点尺寸略微缩小（24px → 20px）
- 连接线更细更淡
- hover提示改为 `滚动到 xx` 而非 `跳转到xx`
- 移动端保持显示（因为它是位置参考）
